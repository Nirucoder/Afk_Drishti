PROBLEM STATEMENT

In rescue and surveillance operations, drones monitor large and inaccessible areas, but manually identifying humans from live video feeds is slow and error-prone.

This can delay emergency response and risk lives, creating the need for an AI system like Drishti with a visual-to-GPS bridge and hybrid intelligence layer that automatically detects humans in real time from drone footage.

IDEA CONCEPT

Autonomous Reconnaissance: Rapidly scans disaster zones faster than ground teams.
Granular Detection: Identifies micro-features like hands or heads from high altitudes.
Noise-Resistant AI: Specialized for "cluttered" environments and structural collapses.
Multi-Frame Logic: Uses temporal verification to ensure 100% detection reliability.
Edge Processing: Onboard computing for real-time, zero-latency detection.
Response Dashboard: Instant data sync for immediate rescue coordination.

TECHNICAL IMPLEMENTATION

The "Nervous System" (Input)

  • Multi-Threaded Pipeline: Ensures the dashboard never freezes, even under heavy load.
  • Universal Input: Seamlessly accepts data from IP cams, USB webcams, or RTSP streams.
  • Asynchronous Nervous System: Decouples video rendering from heavy AI processing.

The Switch

Intelligent Hot-Swap: Instantly detects internet loss and switches from Cloud to Local processing.

Dual-Model Architecture:

  • Online: Leverages Roboflow API for high-accuracy, deep-learning analysis.
  • Offline: Falls back to an optimized YOLOv8 model running on the edge.

Zero Downtime: The transition happens in milliseconds—no restarts, no blackouts.

Tactical Telemetry Layer

Pixel-to-GPS Logic:

Instantly converts 2D video detections into precise, real-world Latitude/Longitude coordinates.

Live Radar Mapping:

Dynamic visualization that plots human locations on a relative grid for spatial awareness.
